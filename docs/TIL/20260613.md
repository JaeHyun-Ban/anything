### Optional은 왜 사용하는가
Optional<Member> findByLoginId(String loginId);에서 Optional은 Member객체가 있을 수도 있고 없을 수도 있다 는 뜻을 코드로 표현한 타입이다.
이 메서드는 loginId로 회원을 찾고 결과가 있으면 Member를 담은 Optional을 반환한다.
결과가 없으면 null대신 Optional.empty()를 반환한다.

왜 사용하냐면, 가장 큰 이유는 NullPointerException을 줄이고, "값이 없을 수 있음"을 명확하게 드러내기 위해서이다.
```java
Optional<Member> result = memberRepository.findByLoginId("abc");

if (result.isPresent()) {
    Member member = result.get();
}

Member member = memberRepository.findByLoginId("abc")
        .orElseThrow(() -> new IllegalArgumentException("회원을 찾을 수 없습니다."));
```

### @AllArgsConstructor와 @Builder의 차이점

@AllArgsContructor는 모든 필드를 받는 생성자를 만들어 준다.
```java
public Member(Long id, String loginId, String password, String name) {
    this.id = id;
    this.loginId = loginId;
    this.password = password;
    this.name = name;
}
```
이런 생성자가 생긴것 과 같다. 
사용할 때는 순서를 정확히 맞춰야 한다.

```java
Member member = new Member(1L, "user1", "1234", "홍길동");
```
여기서 문제는 필드가 많아지면 값의 의미를 파악하기 어렵고, 순서를 실수하기 쉽다는 점이다.

@Builder는 빌더 패턴으로 객체를 생성하게 해준다.
```java
Member member = Member.builder()
        .loginId("user1")
        .password("1234")
        .name("홍길동")
        .build();
```
장점은 어떤 필드에 어떤 값이 들어가는지 명확하다는 것이다. 그리고 픽ㄹ요한 값만 선택적으로 넣을 수도 있다.

즉
@AllArgsConstructor는 “모든 필드를 받는 생성자를 자동으로 만들기”이고,
@Builder는 “필드 이름을 지정하면서 읽기 좋고 안전하게 객체 만들기”입니다.

### @NotBlank

문자열 값이 비어있으면 안 된다는 검증 어노테이션이다.
@NotBlank는 다음의 값을 허용하지 않는다.
```java
null
""
"   "
```
아예 값이 없거나, 빈 문자열이거나, 공백만 문자열이면 검증에 실패한다.

```java
@PostMapping("/signup")
public ResponseEntity<?> signup(@Valid @RequestBody SignupRequest request) {
    // 회원가입 처리
}
```
컨트롤러에서 위와 같이 사용한다.
@Valid가 있어야 SignupRequest안의 @NotBlank검증이 실제로 실행된다.

## SecurityConfig.java분석

### @Configuration
이 클래스가 스프링 설정 클래스라는 뜻이며 스프링이 이 앱을 실행할 때 이 클래스를 읽고 설정을 등록한다.

### @EnableWebSecurity
Spring Security를 활성화한다는 뜻. 이게 있으면 스프링 시큐리티가 웹 요청에 보안 필터를 적용한다.

```java
@Bean
public PasswordEncoder passwordEncoder() {
    return new BCryptPasswordEncoder();
}
```
비밀번호 암호화 도구를 스프링 빈으로 등록하는 코드이며 회원가입할 때 비밀번호를 암호화할 때 사용한다.
```java
String encodedPassword = passwordEncoder.encode(request.getPassword());
```
