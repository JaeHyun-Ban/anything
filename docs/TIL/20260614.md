## Spring Security
학습 한 내용의 요약은 다음과 같다.
```text
Spring Security를 켠다.
비밀번호는 BCrypt로 암호화한다.
CSRF는 끈다.
서버 세션은 사용하지 않는다.
회원가입과 로그인 API는 누구나 접근 가능하다.
그 외 모든 API는 인증된 사용자만 접근 가능하다.
```

### 코드와 함께 풀이 

```java
@Bean
public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
```
이 메서드는 보안 필터체인 설정이다.

서버로 요청이 들어오면 컨트롤러에 바로 가는게 아니라,Spring Security의 여러 필터를 먼저 거친다.
여기서 "CSRF 켤지", "로그인 없이 접근 가능한 URL은 무엇인지", "세션은 사용할 지"같은 설정을 할 수 있다.

```java
http
    .csrf(AbstractHttpConfigurer::disable)
```
이 코드는 CSRF보고 기능을 끈다는 뜻이다.

CSRF는 웹 브라우저 기반 로그인 세션을 사용할 때 중요한 보안 기능이다.
그런데 REST API서버에서 JWT같은 토큰 방식을 쓰거나, 세션을 사용하지 않는 구조라면 보통 꺼두는 경우가 많다.
즉 이 코드는 "CSRF 검사는 하지 않겠다"는 설정이다.

```java
.sessionManagement(sm -> sm.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
```
이 부분은 세션을 사용하지 않겠다는 뜻이다.
"STATELES"는 서버가 로그인 상태를 세션에 저장하지 않는다는 뜻이다.
일반적인 세션 로그인의 흐름은 다음과 같다.
```java
로그인 성공
-> 서버가 세션 생성
-> 브라우저가 세션 쿠키 저장
-> 다음 요청 때 쿠키로 로그인 상태 확인
```
그런데 "STATELESS"는 서버가 이런 로그인 상태를 저장하지 않는다.
요청마다 토큰 같은 인증 정보를 직접 보내야한다.
그래서 JWT 로그인 방식에서 자주 사용된다.

```java
.authorizeHttpRequests(auth -> auth
        
```
URL별 접근 권한 설정이다.

```java
.requestMatchers("/api/member/signup", "/api/members/login").permitAll()
```
이 두 주소는 로그인하지 않아도 접근 가능하다는 뜻이다

```java
.anyRequest().authenticated()
```
위에서 허용한 URL을 제외한 나머지 모든 요청은 인증이 필요하다는 뜻이다.<br/>
예를 들어 다음과 같은 요청들은 로그인 또는 인증 정보가 있어야 접근할 수 있다.
```java
/api/members/me
/api/posts
/api/comments
```

```java
return http.build();
```
위에서 설정한 내용을 바탕으로 최종 "SecurityFilterChain"객체를 만들어서 스프링에 등록한다.

---

```java
@RestController
```
이 클래스가 REST API컨트롤러라는 뜻이다.<br/>
컨트롤러는 클라이언트의 요청을 받는 입구이다.<br/>
@RestController는 아래 두 개를 합친 것과 비슷하다
```java
@Controller
@ResponseBody
```
즉, 메서드가 반환하는 객체를 HTML페이지가 아니라 JSON 응답으로 변환해준다.<br/>
예를들어 MemberResponse객체를 반환하면 Spring이 자동으로 JSON으로 바꿔준다.

---

```java
@RequiredArgsConstructor
```
final이 붙은 필드를 받는 생성자를 자동으로 만들어 준다.<br/>
```java
@RequiredArgsConstructor
public class MemberController {
    private final MemberService memberService;
    
    // 아래 생성자를 어노테이션이 자동으로 만들어준다.
    public MemberController(MemberService memberService) {
        this.memberService = memberService;
    }
}

```

```java
public ResponseEntity<MemberResponse> signup(@Valid @RequestBody SignupRequest request){
```
### @Valid
SignupRequest안의 검증 조건을 실행한다.
예를들어 loginId, password, namedp @NotBlank가 있으면 빈 값인지 검사한다.

### @RequestBody
HTTP 요청 본문에 들어온 JSON 데이터를 자가 객체로 바꿔준다.

---

### @AuthenticationPrincipal UserDetails userDetails

@AuthenticationPrincipal은 현재 로그인한 사용자 정보를 꺼내오는 어노테이션이다.<br/>
JWT인증이 성공하면 Spring Security안에 "현재 로그인한 사용자" 정보가 저장된다.<br/>
그 정보를 이 파라미터로 바로 받을 수 있게 해주는 어노테이션이다.<br/>
UserDetails는 Spring Security에서 사용하는 사용자 정보 객체이다.
